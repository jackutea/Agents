# Windows OS Local Skill

此 skill 专注于当前 Windows 系统本地操作与配置，实现 Windows 系统级任务执行的规范与模板。

## 核心职责
- 执行本地 Windows 系统配置任务
- 管理计划任务、环境变量、DNS、hosts、SSH 密钥等
- 下载并安装软件，安装前必须询问用户
- 打开软件、关闭软件
- 监控某个端口的流量
- 打印一次系统状态
- 监控系统状态
- 执行键盘操作（按下、按住、弹起特定按键）
- 执行鼠标操作（按下、按住、弹起特定按键，移动鼠标）
- 支持文件下载、截图、系统信息查询等操作
- 提供安全执行建议，避免破坏现有 Windows 环境

## 实现流程
1. 确认当前正在操作的是目标本机 Windows 系统
2. 询问并确认用户要执行的具体操作与必要参数
3. 采用 Windows 系统命令执行任务
4. 如果模型内置能力不足，优先调用 tool 内的 `cmd`
5. 如果 tool 内没有合适的 `cmd`，优先编写 `cmd` 并放入 tool
6. 如果写不出 `cmd`，编写 C++ 程序调用 OS 级接口，编译成 exe 放入 tool 目录，随后再编写 `cmd`
7. 如果依然无法通过 `cmd` 完成，调用“命令提示符”执行
8. 如果“命令提示符”无法完成，则调用 PowerShell
9. 如果依然无法完成任务，告知用户无法完成，并终止任务

## 常见执行能力
- 计划任务：使用 `schtasks`
- 音量调整：通过 命令提示符 脚本或系统接口修改播放音量
- DNS 设置：配置网络适配器的 DNS 服务器地址
- hosts 配置：编辑 `C:\Windows\System32\drivers\etc\hosts`
- SSH Key 配置：创建/安装 `~\.ssh\id_rsa`、`id_rsa.pub` 和 `config`
- PATH 配置：修改当前用户或系统环境变量中的 `Path`
- 文件下载：使用 `Invoke-WebRequest` 或 `curl` 下载文件
- 软件安装：按用户确认后下载安装程序并执行静默安装或可交互安装
- 打开软件：通过 `Start-Process` 或 Shell 命令启动程序
- 关闭软件：使用 `Stop-Process` 或 `taskkill` 终止进程
- 端口流量监控：使用 `Get-NetTCPConnection`、`netstat` 网络性能计数器
- 系统状态打印：收集 CPU、内存、磁盘、网络、进程等信息并输出摘要
- 系统状态监控：持续或定期检查系统指标并报告异常
- 键盘操作：使用 命令提示符 脚本或辅助工具模拟按键事件
- 鼠标操作：使用 命令提示符 脚本或辅助工具模拟鼠标移动和按键事件
- 截图：使用 命令提示符 或辅助工具生成屏幕截图文件

## 约束
- 修改系统文件前需确认目标路径和备份策略
- 不要直接覆盖用户已有重要配置，必要时先创建备份
- 使用中文输出
